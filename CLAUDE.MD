## Commands

- `npx expo start` ‚Äî start dev server (press `a` for Android, `i` for iOS, `w` for web)
- `npx expo lint` ‚Äî run ESLint
- No test runner is configured.

## Architecture

Expo 54 / React Native 0.81 app using expo-router (file-based routing) with TypeScript.

### Routing (`app/`)
- `app/(tabs)/` ‚Äî Bottom tab screens: Home (`index.tsx`), Explore, Schedule, Profile
- `app/admin/` ‚Äî Protected admin section (auth-gated via `_layout.tsx` redirect)
- `app/auth/login.tsx` ‚Äî Login screen
- `app/details/` ‚Äî Detail screens for mosque, event, school info

### State (`lib/stores/`)
Zustand stores, each fetching from a corresponding Supabase table:
- `mosquesStore` ‚Üí `mosques` table
- `eventsStore` ‚Üí `events_activities` table
- `islamicSchoolsStore` ‚Üí `quran_schools` table
- `adhanStore` ‚Äî prayer time calculations (uses `adhan` library + device location)
- `scheduleStore` ‚Äî user schedule/alarms
- `authStore` ‚Äî Supabase auth session, sign-in/out, mosque ownership

### Data
- **Supabase** (`lib/supabase.ts`) ‚Äî all data comes from Supabase with RLS (public SELECT)
- `constants/mockData.ts` ‚Äî TypeScript interfaces (`Mosque`, `Activity`, `IslamicSchool`)

### i18n (`lib/i18n/`)
Three languages: Arabic (`ar.json`), French (`fr.json`), English (`en.json`). Default language is Arabic. All user-facing strings must go in these JSON files and be accessed via `useTranslation()`. RTL is handled via `I18nManager`.

### Theming
- `constants/theme.ts` ‚Äî `Colors` (light/dark), `Spacing`, `BorderRadius`, `Shadows`, `Fonts`
- `context/ThemeContext.tsx` ‚Äî theme provider with `useTheme()` hook
- Always use theme tokens from `constants/theme.ts`; never hardcode colors

### Key Libraries
- `@shopify/flash-list` for performant lists
- `react-native-reanimated` for animations
- `@gorhom/bottom-sheet` for bottom sheets
- `expo-image` for images (not `<Image>` from react-native)
- `adhan` for prayer time calculations
- Path alias: `@/*` maps to project root

## Conventions
- Solve problems with maintainability in mind
- All user-facing text must be in `lib/i18n/*.json` files, never hardcoded
- Admin screens live under `app/admin/`; public screens under `app/(tabs)/` and `app/details/`
